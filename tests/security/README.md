# Security Tests

E2E specs for auth boundaries, permission checks, CSP, or other security-related behavior.

**Add here when:** You introduce dedicated security specs (e.g. unauthenticated redirect, role-based access, CSP headers). Until then, auth flows are covered in `tests/e2e/auth.e2e.test.ts`.

**Tool:** Playwright. Run with `pnpm test:e2e` (or target this directory if you add a separate script).
